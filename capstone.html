The Brief

You are applying for a Senior Frontend Developer role. They have given you a take-home assignment.

Project: "TaskMaster Pro"

Build a Kanban-style task management board (like Trello, but simpler).

Level 1: The MVP (Minimum Viable Product)

  • Columns: Three fixed columns: "To Do", "In Progress", "Done".
  • Tasks:
    • Create a new task (Title + Description).
    • Move tasks between columns (Buttons "Move Right" / "Move Left" are fine).
    • Delete a task.
  • State Management: Use Zustand for the global board state.
  • Persistence: Tasks must survive a page refresh (localStorage).

Level 2: Senior Requirements

  • Drag & Drop: Implement dnd-kit or react-beautiful-dnd to move tasks between columns.
  • Routing: Clicking a task opens a modal or a separate page (/task/:id) to view details. The URL must change so it can be shared.
  • Testing: Write unit tests for the Task Card component and integration tests for the "Add Task" flow.
  • TypeScript: Strict typing. No any. Define interfaces for Task, Column, and BoardState.
  • Simulated Latency: Add a fake delay (500ms) to your store actions. Implement Optimistic UI (update the UI immediately) or Loading States (spinners) to handle this gracefully.

Level 3: The "Hired" Features

  • Performance: Use React.memo on the TaskCard component. Ensure that dragging one card doesn't re-render the other 50 cards. (Verify with React DevTools Profiler).
  • Accessibility: Ensure the Drag & Drop is keyboard accessible (Space to lift, Arrows to move, Space to drop).
  • Dark Mode: Implement a theme switcher that persists.

Evaluation Criteria (Self-Check)

  1. Did you use Zustand?
  2. Is your file structure clean? (Components folder, Hooks folder, Types folder).
  3. Are there any console errors? (There should be zero).
  4. Do your tests pass?
  5. Did you use absolute imports?

Final Step

Once you build this, delete it. And build it again without looking at your old code. That is how you master it.

Good luck. You are ready.
