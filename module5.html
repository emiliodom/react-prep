1. Core Concepts

Custom Hooks: JavaScript functions that start with use... and call other hooks. They allow you to share logic (like fetching or form handling) between components without sharing UI.

useReducer: An alternative to useState for complex logic. You dispatch "actions" (instructions) to a "reducer" (function) that calculates the new state.

Memoization (useMemo/useCallback): Performance tools. They tell React: "If these inputs haven't changed, don't re-calculate this value or re-create this function."

1.1 Deep Dive: The Rules of Hooks

Hooks are magic, but they have rules. They rely on call order.
1. Only call hooks at the top level. Never inside loops, conditions, or nested functions.
2. Only call hooks from React functions.

1.2 Industry Standard: The "use" Prefix

If a function uses a hook, it must be named useSomething. This is not just a convention; linters rely on it to enforce the Rules of Hooks. If you write a helper function that calls useContext but name it getAuth(), you will break the linter and potentially your app.

2. The Build Challenge: "Custom Hook Library"

Level 1: The Basics

Extract logic from your previous modules into reusable hooks.

  1. useFetch(url):
    • Accepts a URL.
    • Returns { data, loading, error }.
    • Handles the AbortController logic internally.
  2. useLocalStorage(key, initialValue):
    • Syncs a state variable with localStorage.
    • Updates storage when state changes.
    • Initializes state from storage if it exists.

Implementation: Use useFetch in your Users page. Use useLocalStorage to persist the "Dark Mode" preference (add a toggle for this).

Level 2: Senior Constraints (Generics)

  • Typed Hooks: Your useFetch must be generic: useFetch<User[]>(...). It should not return any.
  • Typed Storage: Your useLocalStorage should infer the type from the initial value.
  • Event Listeners: Create useWindowSize(). It must attach a 'resize' listener on mount and remove it on unmount. If you forget the cleanup, you fail.

Level 3: Nice to Haves

  • Debounce Hook: Create useDebounce(value, delay). Use this for the search bar in Module 3. It should only trigger the API search 500ms after the user stops typing.
  • Previous Value: Create usePrevious(value) to track what a prop was in the last render. Useful for comparing changes.

3. The Bug Hunt

üêõ Scenario: The Expensive Calculation

The Setup: Create a function that is intentionally slow (e.g., a loop that runs 10 million times). Call this function inside your component body to calculate a derived value.

The Issue: Every time you type in an unrelated input field, the UI freezes because the component re-renders and re-runs the slow function.

The Task: Wrap the calculation in useMemo so it only runs when its specific dependencies change.

4. Test on your own

Verify your Hooks

  • Storage Sync: Open your app in two tabs. Change the "Dark Mode" toggle in Tab 1. Does Tab 2 update automatically? (Hint: Listen for the 'storage' event in your hook).
  • Debounce Check: Type "React" quickly in your search bar. Check the Network tab. Did you see 5 requests (R, Re, Rea...) or just 1 request at the end? (You want just 1).
